# 13. SparkFest Submission Checklist

Cross-referenced against official SparkFest 2026 Elimination Round requirements. Use this as the final go/no-go list before July 2 submission.

## 13.1 Elimination Round Deliverables

- [ ] Project Brief written and reviewed
- [ ] 3-Minute Video Presentation recorded and edited
- [ ] Public GitHub Repository — set to public, complete README, install/usage instructions, visible commit history throughout the build
- [ ] One-Page Project Document (PDF) finalized
- [ ] Working Prototype or MVP — installable APK or accessible demo build
- [ ] Supporting materials (optional): pitch deck, research citations, screenshots

## 13.2 GitHub Repository Requirements

- [ ] Repository set to Public
- [ ] README includes: problem statement, solution overview, tech stack, setup instructions, team members
- [ ] Regular commits throughout the build period (avoid one large final upload — this is explicitly flagged as suspicious per competition rules)
- [ ] At least one Google Technology clearly integrated and documented (Gemini API + Firebase)
