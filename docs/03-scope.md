# 3. Scope: MVP Boundaries

Given the 4-day build window (June 28 – July 2), scope is deliberately constrained. Anything not listed under "In Scope" should be treated as a post-hackathon roadmap item, not a V1 deliverable.

## 3.1 In Scope (V1 / MVP)

- Senior profile creation (assisted registration by family/volunteer)
- Trusted Circle setup — add 1 or more family/volunteer members per senior
- Daily check-in flow (mood, activity, who they talked to) — large-button, low-text UI
- Gemini-generated weekly companionship summary, viewable by Trusted Circle
- Scam message/number checker — paste or forward suspicious text, get a plain-language verdict
- Real-time push alert to Trusted Circle when a scam is flagged or a check-in is missed for 2+ days
- Basic OSCA/barangay contact directory (static data, Laguna-focused for demo)

## 3.2 Out of Scope (V1)

- Live call recording / call screening (Phonely-style) — requires deeper OS-level permissions, not feasible in 4 days
- Gamified rewards/points redemption system (TANDA-NAN style) — V2 candidate
- Multi-language support beyond Filipino/English/Taglish
- Government system integration (live OSCA/DSWD APIs) — use static/mock data for demo
- Offline-first full sync — basic offline tolerance only, not a core requirement
